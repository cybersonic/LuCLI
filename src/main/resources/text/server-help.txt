LuCLI Server Management

Usage: lucli server [command] [options]

Commands:
  start [options]            Start a Lucee server for the current directory
    --version VERSION        Specify Lucee version
    --name NAME              Specify custom server name
    --force                  Replace existing server
    --env NAME               Apply environment overrides from lucee.json
    --dry-run                Preview resolved configuration without starting
    --include LIST           Comma-separated sections: config,env,lucee,tomcat-web,tomcat-server,https-keystore-plan,https-redirect-rules,all
    --include-env            Include environment preview section (same as --include env)
    --include-all            Include all dry-run sections (same as --include all)
    --include-lucee          Include Lucee CFConfig dry-run preview
    --include-tomcat-web     Include generated web.xml dry-run preview
    --include-tomcat-server  Include generated server.xml dry-run preview
    --include-https-keystore-plan
                             Include HTTPS keystore dry-run plan
    --include-https-redirect-rules
                             Include HTTPS redirect rules dry-run plan
    --disable-lucee          Start as static server (enableLucee=false for this start)
    --enable-lucee           Explicitly enable Lucee CFML engine for this start
    --warmup                 Enable Lucee warmup mode for this start
    --prewarm                Download runtime artifacts and exit without starting
    --agents LIST            Comma-separated agent IDs to enable for this start
    --enable-agent NAME      Enable a specific agent for this start
    --disable-agent NAME     Disable a specific agent for this start
    --no-agents              Disable all agents for this start
  run [options]              Run a Lucee server in foreground (Ctrl+C to stop)
    --version VERSION        Specify Lucee version
    --name NAME              Specify custom server name
    --force                  Replace existing server
    --env NAME               Apply environment overrides from lucee.json
    --dry-run                Preview resolved configuration without running
    --include LIST           Comma-separated sections: config,env,all
    --include-env            Include environment preview section (same as --include env)
    --disable-lucee          Run as static server (enableLucee=false for this run)
    --enable-lucee           Explicitly enable Lucee CFML engine for this run
    --warmup                 Enable Lucee warmup mode for this run
    --prewarm                Download runtime artifacts and exit without starting
    --agents LIST            Comma-separated agent IDs to enable for this run
    --enable-agent NAME      Enable a specific agent for this run
    --disable-agent NAME     Disable a specific agent for this run
    --no-agents              Disable all agents for this run
  stop [options]             Stop the server for the current directory
    --name NAME              Stop a specific server by name
    --config FILE            Resolve server name from an alternate config file
  status [options]           Show server status for the current directory
    --name NAME              Show status for a specific server by name
    --config FILE            Resolve server name from an alternate config file
  list                       List all server instances
  prune [options]            Remove stopped server instances
    --all                    Remove all stopped servers
    --name NAME              Remove a specific stopped server by name
    --config FILE            Resolve server name from an alternate config file
    --force                  Skip confirmation and attempt stop before pruning
  monitor [options]          Monitor a Lucee server via JMX
  log [options]              View server logs (tomcat, server, web)

Configuration:
  Server configuration is read from lucee.json in the current directory.
  If lucee.json doesn't exist, a default one will be created.
  If multiple server slugs are associated with the same project path,
  use --name (or --config where supported) to disambiguate project-scoped commands.

Examples:
  lucli server start                            # Start server with default settings
  lucli server start --version 6.1.0.123        # Start server with specific version
  lucli server start --agents luceedebug        # Start with only the luceedebug agent enabled
  lucli server start --enable-agent luceedebug  # Enable luceedebug in addition to default agents
  lucli server start --prewarm                  # Download runtime artifacts only (no start)
                                                
  lucli server start --no-agents                # Start with all agents disabled
  lucli server stop                             # Stop the server for current directory
  lucli server stop --name my_server            # Stop a specific server by name
  lucli server status                           # Check server status for current directory
  lucli server status --name my_server          # Check status for a specific server
  lucli server list                             # List all servers
  lucli server prune                            # Remove stopped server for current directory
  lucli server prune --all                      # Remove all stopped servers
  lucli server monitor                          # Monitor server via JMX dashboard
